# 故障排查（现象 -> 原因 -> 处理）

## 1) 服务不启动
- 现象：`launchctl print` 显示未运行。
- 原因：启动命令错误、env 缺失、依赖未安装。
- 处理：先看 `stderr.log` 首条错误，再修正命令或变量。

## 2) 端口被占用
- 现象：服务启动失败或日志提示 address already in use。
- 原因：其他进程已监听同端口。
- 处理：改 `PORT`，重启服务并复查监听。

## 3) 权限错误
- 现象：日志提示 permission denied。
- 原因：服务用户对目录或文件无权限。
- 处理：修正属主与权限，确保服务用户可读写目标目录。

## 4) Provider 不可达（中国网络）
- 现象：请求超时、连接失败。
- 原因：网络限制或目标 API 不可达。
- 处理：切换国内可达 Provider，或在需要时启用 VPN。

## 5) 渠道消息收不到
- 现象：机器人不回消息。
- 原因：Token 错误、Webhook 未生效、渠道配置不完整。
- 处理：重填 token，复核渠道配置，查看渠道回调日志。

## 6) 重启后未自动拉起
- 现象：开机后服务未运行。
- 原因：LaunchDaemon 未正确 bootstrap/enable。
- 处理：重新 bootstrap + enable，检查 plist 权限（root:wheel 644）。

## 7) 配置改完不生效
- 现象：行为没有变化。
- 原因：未重启服务或改错配置文件路径。
- 处理：确认修改的是生产 env，执行 kickstart 重启。

## 8) 日志刷屏重启
- 现象：日志重复同一错误，服务不断拉起。
- 原因：启动即崩溃（变量缺失、命令错误、依赖故障）。
- 处理：先修复根因再重启，必要时临时停服务避免刷屏。

## 9) 进程属主不对
- 现象：`ps` 显示不是 `svc_openclaw`。
- 原因：以错误用户手工启动或 plist 配置 UserName 不正确。
- 处理：停止错误进程，修正 plist 后按 launchd 方式重启。

## 10) 升级后异常
- 现象：升级后启动失败或功能异常。
- 原因：依赖变更、配置项变化。
- 处理：回滚到上一 commit/tag，恢复备份 env，再逐项比对变更。
